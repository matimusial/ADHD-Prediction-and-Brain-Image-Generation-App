Instructions for the Administrator

1. Familiarization with the Code Before Running the Program

Before starting work with the application, it is recommended to become familiar with the project's source code. The user interface (UI) is a demonstration version, and full functionality of the program is available through direct interaction with the code. If you plan to perform more advanced operations, it is better to use the code directly rather than relying on the UI.

2. Access to the Model Database and Management

The administrator has full access to the model database. This allows for quick browsing and deletion of models without the need for additional database management applications.

3. Running CNN Training for MRI Data

The administrator can run CNN (Convolutional Neural Network) training for MRI data, but only for files located in the mri_real_files folder. To correctly run the training, appropriate files must be prepared in advance. The project is optimized for data shaped as 120x120x1 and has not been tested with other sizes.

From the user interface, it is possible to adjust parameters such as the number of epochs, batch size, learning rate, and test size. You can also add your own description of the training. After completing the training, there is an option to easily add the newly trained model to the database. During training, metrics are displayed on the UI graph.

4. Handling Files for CNN

For CNN training, the administrator can work with three types of files: CSV, EDF, and MAT. The core of the project is based on MAT files, and the other formats have been adapted to match the MAT format.

If the administrator wants to train the model on another group of patients with files in different formats, they can use conversion functions available in the helpers_unused.py file. It is recommended to review the read_eeg_raw function, which prepares the data for further processing. The project was trained on data sized 19x128, and the data frame size has only been tested superficially.

To correctly load files through the UI, the user must select a folder that contains two subfolders: adhd (data for patients with ADHD) and control (data for healthy patients), with the respective EEG files.

5. GAN Training

The training of GAN (Generative Adversarial Network) models is conducted similarly to CNN training for MRI data. During the training, graphs and images from the network's generator are generated in the interface. It is worth noting that for GAN models, the generator loss is the average of all the losses collected during training, with each loss being recorded every 5% of the total epochs.

6. UI Usage vs. Code Operation

The user interface was designed as a superficial support tool. For more advanced operations, the administrator should work directly with the project code, using the UI only as a supportive aid.

7. Hardware Resource Consideration

Please note that training processes, especially for GAN models, are very resource-intensive and may take a long time, especially with large datasets. It is recommended to run these processes on professional machine learning hardware with adequate memory and computational power. Otherwise, training may significantly slow down the system.

8. GAN Training Warning

When starting GAN training, a warning message appears stating that generating synthetic data using GAN models can be time-consuming, particularly for large datasets. Make sure that your system has enough resources and patience for the process to complete.

Instructions for the Doctor

1. Loading MRI and EEG Data

The doctor can load data using the "Load data" button at the top of the screen. The supported formats are:

  • MRI: .nii format,
  • EEG: .mat, .csv, and .edf formats.

All loaded data must correspond to the formats used by the administrator to train and process the models. The "MRI models" and "EEG models" sections allow the selection of the model to use for prediction, where each model name corresponds to its accuracy.

2. Checking Multiple Files Simultaneously

The doctor can load and check multiple MRI or EEG files at once using the appropriate fields: "MRI files" or "EEG files". The prediction result will be the average of all selected files.

3. Selecting a Model and Making Predictions

After loading the files, select the appropriate model from the "MRI models" or "EEG models" lists. The chosen model will appear in the fields "Chosen model EEG" or "Chosen model MRI". Click the "Predict" button to start the prediction process. The results will be displayed in the appropriate tabs:

  • EEG tab: EEG signals will be displayed as plots,
  • MRI tab: MRI images will be displayed with options to switch planes and files using the navigation buttons.

4. Viewing MRI Images Used for Model Training

The left panel provides additional options:

  • "Show Real Brain Examples": Allows the doctor to view real MRI images used for training CNN or GAN models.
  • "Show Generated Brain Examples": Enables the display of MRI images generated by the best GAN model, allowing a comparison between real and generated images.

5. Generating New Brain Images

The "Generate New Brain Image" option allows the doctor to generate new brain images from the models stored in the database. Each generated image can be saved individually to a file.

6. Additional Options

Once the task is completed, the doctor can switch back to the admin view using the "Admin View" button or exit the application by clicking the "Exit" button.
